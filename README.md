Scry predicts infrastructure failure states from a stream of metrics. It sorts each resource into one of five states (healthy, needs capacity soon, heading for failure, degrading, or anomalous) and forecasts where the metrics are headed.

No trained models are included. You bring your own metrics as a table (one row per resource, metric, timestamp, and value) from a local file or object storage, then train your own. LogicMonitor is supported as a data source.
